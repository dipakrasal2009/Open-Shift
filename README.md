## 1. Prerequisites

- **Red Hat Account** – Required to download the pull secret.
- **AWS CLI** – Installed and configured on your Linux EC2 instance.

---

## 2. Domain and DNS Preparation

### 2.1 Purchase a Domain

Buy a domain from any provider (e.g., GoDaddy, Namecheap, AWS Route 53 Registrar).

OpenShift requires a routable domain because it creates:

| Endpoint | Format |
|---|---|
| API endpoint | `api.<cluster-name>.<base-domain>` |
| Application routes | `*.apps.<cluster-name>.<base-domain>` |

### 2.2 Create a Hosted Zone in AWS Route 53

Inside AWS Route 53:
- Create a **public hosted zone** using the purchased domain.
- Copy the **NS (Name Server) records** generated by Route 53.

### 2.3 Add NS Records to Domain Provider

In your domain provider's portal, replace the existing NS records with the Route 53 NS values.

## 3. Prepare an EC2 Instance for Installation

### 3.1 Launch a Free-Tier EC2 Instance

Use any Linux distribution (Amazon Linux, RHEL, or Ubuntu). This instance will:
- Run the OpenShift installer.
- Communicate with AWS to provision cluster resources.

### 3.2 Switch to Root

```bash
sudo su -
```

### 3.3 Verify and Configure AWS CLI

```bash
aws --version
aws configure
```

Provide when prompted:
- AWS Access Key ID
- AWS Secret Access Key
- Default region (e.g., `us-east-1`)
- Output format: `json`

---

## 4. Generate SSH Keys

```bash
ssh-keygen -t ed25519 -N '' -f ~/.ssh/id_rsa
```

This allows OpenShift to securely access the cluster machines during installation and debugging.

---

## 5. Download OpenShift Install Tools

Create a working directory:

```bash
mkdir openshift
cd openshift
```

Download the installer and client:

```bash
wget https://mirror.openshift.com/pub/openshift-v4/clients/ocp/stable/openshift-install-linux.tar.gz
wget https://mirror.openshift.com/pub/openshift-v4/clients/ocp/stable/openshift-client-linux.tar.gz
```

Extract both archives:

```bash
tar -xzf openshift-install-linux.tar.gz
tar -xzf openshift-client-linux.tar.gz
```

| File | Purpose |
|---|---|
| `openshift-install-linux.tar.gz` | The installer that provisions and builds the cluster on AWS |
| `openshift-client-linux.tar.gz` | Provides the `oc` CLI to interact with the cluster after creation |

## 6. Generate Installation Configuration

```bash
./openshift-install create install-config --dir=/root/openshift
```

You will be prompted for:

| Prompt | Value |
|---|---|
| SSH Public Key | Path to your public key |
| Platform | `aws` |
| Region | e.g., `us-east-1` |
| Base Domain | Your purchased domain |
| Cluster Name | e.g., `mycluster` |
| Pull Secret | From [Red Hat Console](https://console.redhat.com/openshift/install/aws/user-provisioned) |

This generates `install-config.yaml`. You may edit it to change:
- Number of master nodes
- Number of worker nodes

> **Tip:** Back up `install-config.yaml` before running the installer — it gets consumed during cluster creation.

---

## 7. Create the OpenShift Cluster

> Administrative AWS permissions are required.

```bash
./openshift-install create cluster --dir=/root/openshift --log-level=info
```

Cluster creation takes **40–60 minutes**. AWS resources created include:

- VPC
- NAT Gateways
- Subnets
- Security Groups
- Bootstrap Node
- Master Nodes (3)
- Worker Nodes (2+)
- Load Balancers
- Route 53 DNS Records
- Elastic IPs

---

## 8. Set Up Kubeconfig to Access the Cluster

```bash
export KUBECONFIG=/root/openshift/auth/kubeconfig
echo 'export KUBECONFIG=/root/openshift/auth/kubeconfig' >> $HOME/.bashrc
```

This ensures all `oc` CLI commands target your new cluster.

---

## 9. Verify Cluster Operation

**List all nodes:**
```bash
oc get nodes
```

**Get the console admin password:**
```bash
cat auth/kubeadmin-password
# Username: kubeadmin
```

**Get the console URL:**
```bash
oc get routes -n openshift-console | grep console
```

Open the URL in a browser and log in with `kubeadmin` and the password from above.

---

## 10. Destroy the Cluster

```bash
./openshift-install destroy cluster --dir=/root/openshift --log-level=info
```

This removes all AWS resources created during installation:
VPC, NAT Gateways, Subnets, Security Groups, Bootstrap/Master/Worker Nodes, Load Balancers, Elastic IPs.

> **Warning:** This action is irreversible. Ensure you have backed up any important data before running this command.
